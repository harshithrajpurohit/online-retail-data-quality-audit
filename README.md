# online-retail-data-quality-audit
Retail Revenue Reconciliation &amp; Invoice Quality Audit

# Project Overview

A UK-based online retailer reported discrepancies between the revenue generated by its internal reporting system and the figures reported by the finance team.

This project investigates the underlying causes of revenue mismatch using transaction-level invoice data from the UCI Online Retail dataset.

The objective is to build a clean, analysis-ready dataset, identify data quality issues, reconcile revenue calculations, and provide business recommendations to improve reporting accuracy.

# Business Problem

The finance department reported lower monthly revenue than the internal reporting system.

Possible causes include:

Cancelled invoices
Product returns
Duplicate transactions
Invalid pricing
Missing customer IDs
Poor data quality

This project identifies each issue and proposes corrected revenue logic.
